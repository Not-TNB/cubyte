# extension layout

- `include/` public headers shared between extension passes
- `src/` c implementations for the extension compiler pieces
- `docs/` extension-specific grammar and design notes

Start with `docs/using-cubyte.md` for the practical compile-and-run workflow.

`extension_spec.latex` and the generated `extension_spec.md` live at the repo
root as local planning notes and are ignored by git.
